# plan.md

브라우저 기반 음악 제작 웹 앱 개발 계획. 단계(Phase)별로 점진적으로 쌓아 올린다.
개발·디자인 규칙은 `CLAUDE.md` 참고.

핵심 전략: **뼈대를 먼저 세우고(언락·네비·공통 오디오 모듈), 섹션은 가장 단순한 녹음부터 가장 복잡한 DJ 순으로** 하나씩 완성한다. 각 Phase 끝에는 항상 "배포해도 깨지지 않는 상태"를 유지한다.

## Phase 4 — 클럽 디제잉 ✅ 완료 (2026-06-12 검증)

> 순서 변경: 기존 Phase 5(클럽 디제잉)를 Phase 4로 앞당기고, 리믹스 녹음을 Phase 5로 미룸. 드럼 다음으로 "진짜 디제잉 기계" 경험을 먼저 완성한다.

목표: 두 덱 + 믹서로, 실제 2채널 DJ 컨트롤러처럼 두 트랙을 불러와 섞는다.

### 레이아웃

- **데스크탑/가로(≥ md, 1024px 이상 권장)**: 3컬럼 — `덱 A | 믹서(크로스페이더) | 덱 B`. 클럽 부스 믹서를 옆에서 본 배치.
- **모바일 세로**: 화면 폭이 좁아 3컬럼이 비좁으므로, 상단에 `덱 A / 믹서 / 덱 B` 세그먼트 탭(드럼 모드 토글과 동일한 스타일)을 두고 한 번에 한 패널만 전체 폭으로 표시. 상단에 "가로 모드를 권장합니다" 안내 배너(닫기 가능, 차단 아님).
- 모든 슬라이더는 1차 버전에서 **가로형**으로 통일한다(세로 페이더는 터치 스크롤 충돌 위험이 있어 `touch-action` 검증 후 Phase 7에서 비주얼 폴리시로 검토).

### 덱 패널 (A·B 동일 구성, `src/sections/dj/DeckPanel.tsx`)

- [x] **트랙 로드**: 파일 선택 버튼(`audio/*`) → `decodeAudioData` → 트랙명 + 길이 표시. 디코딩 실패 시 인라인 에러 메시지.
- [x] **시크/위치**: 가로 시크바(진행률) + `0:00 / 3:45` monospace 시간 표시. 드래그로 탐색 시 해당 위치가 큐 포인트로 갱신.
- [x] **재생/일시정지 + 큐**: 큰 버튼(≥ 56px). 재생 중 덱 라벨이 강조색으로 점등.
- [x] **너지(임시 피치 벤드)**: ◀ ▶ 버튼을 누르고 있는 동안 ±4% 속도 변화 — 자키휠 대용 수동 비트 보정.
- [x] **피치 페이더**: ±8/±16/±50% 범위 토글 + 가로 슬라이더, 현재 % 와 보정된 BPM을 monospace로 표시, 리셋 버튼.
- [x] **BPM 입력**: 기준 BPM 수동 입력 + "매치" 버튼(다른 덱의 보정 BPM에 맞춰 이 덱의 피치를 자동 계산·적용 — **1회성 보정**이며, 지속적 자동 비트매칭은 아님 → Non-goal과 충돌 없음).
- [x] **EQ 3밴드**: Low/Mid/High 가로 슬라이더 (-24dB ~ +6dB), `Tone.EQ3` 사용, 전체 리셋 버튼.
- [x] **채널 볼륨**: 가로 슬라이더 (0~1).

### 믹서 패널 (`src/sections/dj/MixerPanel.tsx`)

- [x] **크로스페이더**: 가로 슬라이더 (0=A ~ 1=B), 중앙 리셋. `Tone.CrossFade`로 구현.
- [x] 마스터 볼륨은 기존 헤더의 전역 컨트롤을 재사용(중복 컨트롤 만들지 않음).

### 오디오 엔진 (`src/audio/dj/`)

- [x] `deckEngine.ts`: 덱 1개 = `Tone.Player → Tone.EQ3 → Tone.Gain(채널 볼륨)`. 재생 위치는 `Tone.now()` 기준으로 직접 추적(오프셋 + 경과시간 × playbackRate), 피치/시크 시 오프셋 재계산(rebase).
- [x] ~~`useDjDeck.ts` / `useDjMixer.ts`(분리)~~ → **변경**: 덱 A/B + 믹서를 하나의 `src/sections/dj/useDjEngine.ts` 훅으로 통합 — 재생 중에만 `requestAnimationFrame`으로 위치 폴링, 트랙 끝 도달 시 자동 정지, 언마운트 시 모든 노드 `dispose()`까지 한 곳에서 일관되게 관리.
- [x] DJ 덱은 전역 `Tone.Transport`(드럼 시퀀서가 쓰는)와 독립적으로 동작 — 섹션 간 독립성 유지.

### 키보드 단축키

- 덱 A: `A` 재생/일시정지, `S` 큐 · 덱 B: `K` 재생/일시정지, `L` 큐 (드럼 섹션의 1~8과 충돌 없음)

### 1차 범위 밖 (다음 단계로 이월)

- 외부 음원 업로드는 이번엔 **DJ 전용 최소 구현**(파일 선택만)으로 한정. 드래그앤드롭, 무료 음원 라이브러리, 섹션 간 트랙 공유는 Phase 6에서 일반화.
- DJ 믹스 출력 녹음 → MP3 (선택 항목, Phase 5/7 이후 검토)
- 헤드폰 큐(PFL) — 브라우저 다중 오디오 출력 제약(`setSinkId` 미지원 브라우저 존재)으로 백로그
- 회전형 자키휠 비주얼 — v1은 가로 시크바 + 너지 버튼으로 대체

**완료 기준**: 두 트랙을 각 덱에 불러와 재생/큐/피치/EQ/채널 볼륨을 조절하고, 크로스페이더로 두 덱을 자연스럽게 섞을 수 있다. 모바일/데스크탑 모두 콘솔 에러 없이 동작.

**검증 결과**: 데스크탑 Chrome(1280×800)과 모바일 뷰포트(390×844)에서 두 덱에 트랙 로드 → 재생/일시정지/CUE/시크 → 피치 페이더(±8/±16% 범위 전환 포함) → BPM 매치(범위 클램핑 포함) → EQ 3밴드 조절·리셋 → 크로스페이더 조절·중앙 리셋 → 키보드 단축키(A/S/K/L)까지 전 항목 정상 동작. 지원되지 않는 파일(이미지)을 로드했을 때 인라인 에러 메시지가 표시되고 기존 트랙 상태가 보존됨을 확인. 트랙 끝 도달 시 자동 정지 확인. 모바일 세그먼트 탭(덱A/믹서/덱B) 전환과 가로 모드 안내 배너 닫기 정상 동작. `tsc --noEmit` / `eslint` 클린, 전 과정 콘솔 에러 0건(언락 전 표준 AudioContext 경고만 발생). (iOS Safari 실기기 검증은 미수행 — Phase 7 최종 점검 시 재확인)
